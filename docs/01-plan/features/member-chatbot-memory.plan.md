# Plan: 회원별 AI 챗봇 + 사진 업로드 + 장기 기억 시스템

## 1. 개요

### 1.1 기능 요약
회원별로 운영되는 AI 챗봇 인터페이스. 회원이 사진을 올리며 "오늘 강남에서 교육 행사했어"라고 말하면:
1. 사진을 저장하고
2. 대화에서 일시/장소/활동 내용을 자동 추출하여
3. 회원별 롱텀 메모리에 기록

나중에 "지난달 어디서 행사했지?" 같은 질문에 기억에서 찾아 답변.

### 1.2 핵심 가치
- **활동 기록 자동화**: 사진 + 한 줄 메시지만으로 활동 이력 축적
- **장기 기억**: 회원의 모든 활동 히스토리를 AI가 기억하고 검색
- **회원별 격리**: 각 회원의 데이터와 기억은 완전 분리

### 1.3 대상 사용자
- imPD 플랫폼 회원 (유통사/교육 수강생)
- 최PD 본인 (행사/교육 활동 기록)

## 2. 사용자 시나리오

### 시나리오 1: 활동 기록
```
회원: [사진 업로드] "오늘 강남역 근처 세미나실에서 스마트폰 창업 특강했어"
챗봇: "기록했습니다!
  - 날짜: 2026-03-26
  - 장소: 강남역 세미나실
  - 활동: 스마트폰 창업 특강
  - 사진 1장 저장됨

  수고하셨습니다! 이번 달 벌써 3번째 특강이네요."
```

### 시나리오 2: 기억 조회
```
회원: "지난달에 어디서 행사했지?"
챗봇: "2월에 기록된 활동은 총 5건입니다:
  1. 2/3 - 마포구 상암 DMC, 디지털 마케팅 워크숍 [사진 2장]
  2. 2/10 - 서초구 교대역, 소상공인 컨설팅 [사진 1장]
  3. 2/15 - 강남구 코엑스, MBC 인터뷰 촬영 [사진 3장]
  4. 2/22 - 종로구 광화문, 한국환경저널 편집회의
  5. 2/28 - 온라인 ZOOM, 1기 수료식"
```

### 시나리오 3: 통계/리포트
```
회원: "올해 활동 요약해줘"
챗봇: "2026년 1~3월 활동 요약:
  - 총 활동: 18건
  - 교육/특강: 12건
  - 미디어 출연: 3건
  - 미팅/회의: 3건
  - 가장 많이 방문한 장소: 상암 DMC (5회)
  - 사진: 총 42장"
```

## 4. 기술 설계 방향

### 4.1 데이터베이스 스키마

#### `chat_conversations` (대화 세션)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 자동증가 |
| memberId | INTEGER FK → members | 회원 ID |
| title | TEXT | 대화 요약 제목 (AI 자동 생성) |
| createdAt | TIMESTAMP | 생성일 |
| updatedAt | TIMESTAMP | 마지막 메시지 시각 |

#### `chat_messages` (대화 메시지)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 자동증가 |
| conversationId | INTEGER FK | 대화 세션 ID |
| role | TEXT (user/assistant) | 발화자 |
| content | TEXT | 메시지 내용 |
| imageUrls | TEXT (JSON) | 첨부 이미지 URL 배열 |
| createdAt | TIMESTAMP | 전송 시각 |

#### `member_memories` (롱텀 메모리)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 자동증가 |
| memberId | INTEGER FK → members | 회원 ID |
| type | TEXT | activity / note / preference |
| date | TEXT | 활동 날짜 (YYYY-MM-DD) |
| location | TEXT | 장소 (nullable) |
| category | TEXT | 교육, 미디어, 미팅, 행사, 기타 |
| summary | TEXT | 활동 요약 (1줄) |
| detail | TEXT | 상세 내용 |
| imageUrls | TEXT (JSON) | 관련 사진 URL 배열 |
| sourceMessageId | INTEGER FK | 원본 메시지 ID |
| metadata | TEXT (JSON) | 추가 메타데이터 |
| createdAt | TIMESTAMP | 기록 시각 |

#### `member_uploads` (회원 업로드 파일)
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | INTEGER PK | 자동증가 |
| memberId | INTEGER FK → members | 회원 ID |
| filename | TEXT | 원본 파일명 |
| storagePath | TEXT | 저장 경로 |
| fileSize | INTEGER | 파일 크기 (bytes) |
| mimeType | TEXT | MIME 타입 |
| createdAt | TIMESTAMP | 업로드 시각 |

### 4.2 API 라우트

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/chat/conversations` | 회원의 대화 목록 |
| POST | `/api/chat/conversations` | 새 대화 생성 |
| GET | `/api/chat/conversations/[id]/messages` | 대화 메시지 조회 |
| POST | `/api/chat/messages` | 메시지 전송 (AI 응답 포함) |
| POST | `/api/chat/upload` | 사진 업로드 |
| GET | `/api/chat/memories` | 회원의 기억 조회 (검색 지원) |
| GET | `/api/chat/memories/search` | 자연어 기억 검색 |

### 4.3 AI 처리 흐름

```
사용자 메시지 + 사진
  ↓
1. 사진 저장 (member_uploads)
  ↓
2. AI에게 전달:
   - 시스템 프롬프트: "활동 기록 추출 + 대화 응답"
   - 회원의 최근 기억 N건 (컨텍스트)
   - 현재 메시지 + 이미지
  ↓
3. AI 응답:
   - 친근한 대화 응답
   - 구조화된 활동 데이터 (JSON)
     { date, location, category, summary }
  ↓
4. 저장:
   - chat_messages에 대화 저장
   - member_memories에 활동 기록 저장
   - 다음 대화에서 참조 가능
```

### 4.4 페이지 구조

```
/chopd/chat              → 챗봇 메인 (로그인 필요)
/chopd/chat/[id]         → 특정 대화
/chopd/chat/memories     → 나의 활동 기록 타임라인
```

### 4.5 AI 프로바이더
- OpenAI GPT-4o (이미지 인식 + 텍스트 생성)
- 환경변수: `OPENAI_API_KEY`
- 향후 Claude API로 전환 가능

### 5.1 챗봇 UI

## 6. 구현 순서

1. **DB 스키마 추가** — 4개 테이블 + 마이그레이션
2. **파일 업로드 API** — 사진 저장 엔드포인트
3. **챗봇 API** — 대화/메시지 CRUD + AI 연동
4. **메모리 파싱 로직** — AI로 활동 데이터 추출 + 저장
5. **메모리 검색 API** — 회원별 기억 조회/검색
6. **챗봇 UI** — 채팅 인터페이스 컴포넌트
7. **활동 기록 페이지** — 타임라인 뷰
