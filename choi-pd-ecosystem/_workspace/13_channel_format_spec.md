## 2. 채널별 상세 스펙

### 2.1 인스타그램

#### 콘텐츠 유형별 규격

| 유형 | 이미지 크기 | 비율 | 동영상 길이 | 특이사항 |
|------|-----------|------|-----------|---------|
| **피드 (정사각)** | 1080x1080px | 1:1 | - | 기본 형태 |
| **피드 (세로)** | 1080x1350px | 4:5 | - | 가장 많은 화면 차지 (권장) |
| **피드 (가로)** | 1080x566px | 1.91:1 | - | 화면 차지 적음 |
| **캐러셀** | 1080x1350px | 4:5 | - | 최대 20장, 첫 장이 핵심 |
| **스토리** | 1080x1920px | 9:16 | 최대 60초 | 상하 안전영역 250px |
| **릴스** | 1080x1920px | 9:16 | 최대 20분 | 주 4-7개 권장 |

#### 텍스트 규칙
- 캡션 최대: 2,200자
- 더보기 전 노출: **125자** (이 안에 핵심 메시지 + CTA 포함)
- 줄바꿈: 지원 (마침표 뒤 줄바꿈 활용)
- 멘션: @username 형태
- 해시태그: 최대 30개, **권장 5-15개** (피라미드 전략: 대형 3 + 중형 5 + 소형 7)

#### API 연동
- **엔드포인트**: Instagram Graph API (Meta 플랫폼)
- **게시**: `POST /{ig-user-id}/media` → `POST /{ig-user-id}/media_publish`
- **Rate Limit**: 100 포스트/24시간 (롤링 윈도우)
- **확인**: `GET /{ig-user-id}/content_publishing_limit`
- **필요 권한**: Facebook 비즈니스 계정 + Instagram Professional 계정

### 2.2 X (트위터)

#### 텍스트 규칙
- 무료 계정: **280자**
- X Premium: **25,000자**
- 바이오: 160자
- DM: 10,000자
- URL: 실제 길이 무관하게 항상 **23자**로 계산
- 이모지: **2자**로 계산 (Unicode 더블 와이드)

#### 스레드 형식
- 각 트윗 280자 독립 제한
- "1/5" 등 넘버링은 글자 수에 포함
- 이미지/동영상/GIF/투표는 글자 수 미포함

#### 이미지 규격
| 유형 | 크기 | 비율 | 파일 크기 |
|------|------|------|----------|
| **트윗 이미지** | 1200x675px | 16:9 | 웹 15MB / 모바일 5MB |
| **프로필** | 400x400px | 1:1 | 2MB |
| **헤더** | 1500x500px | 3:1 | 2MB |
| **카드** | 1200x628px | 1.91:1 | - |

#### API 연동
- **엔드포인트**: X API v2
- **게시**: `POST /2/tweets`
- **Rate Limit (Basic)**: 100 트윗/24시간
- **미디어 업로드**: 500/24시간/유저
- **전체**: 2,400 트윗/일 (계정 기준)
- **티어별 요금**: Free (읽기만) / Basic ($100/월) / Pro ($5,000/월)

| 항목 | 규격 | 비고 |
|------|------|------|

| **본문 길이** | 1,500-3,000자 (최적) | 600자 미만 시 상위 노출 불리 |

| **이미지 배치** | 본문 3-4줄마다 1장 | 첫 이미지는 본문 상단 |

#### API 연동
- **상태**: 글쓰기 API 존재하나 제한적 (광고성 글 남용으로 2020년 대폭 축소)
- **검색 API**: `GET /v1/search/blog.json` (조회만 가능)
- **대안**: Selenium/Playwright 기반 자동화 (비공식)

### 2.5 워드프레스

#### 콘텐츠 포맷
- **Gutenberg 에디터**: 블록 기반 (HTML + JSON 메타데이터)
- **Classic Editor**: HTML / 비주얼 모드
- **마크다운**: 플러그인 설치 시 지원 (Jetpack Markdown 등)
- **카테고리**: 계층 구조 (부모-자식)
- **태그**: 비계층, 제한 없음

#### Yoast SEO 기준
| 항목 | 권장 규격 |
|------|----------|
| **SEO 제목** | 50-60자 |
| **메타 설명** | 120-155자 |
| **포커스 키워드** | 제목, 첫 문단, 소제목에 포함 |
| **본문 길이** | 1,500-2,500단어 (영문 기준) |
| **가독성** | Flesch Reading Ease 60+ |
| **내부 링크** | 최소 1개 이상 |
| **이미지 alt 텍스트** | 포커스 키워드 포함 |

#### API 연동
- **엔드포인트**: `POST /wp-json/wp/v2/posts`
- **인증**: JWT / Application Passwords / OAuth 2.0
- **Rate Limit**: 기본 제한 없음 (호스팅 환경 의존)
- **Yoast 메타 설정**: `_yoast_wpseo_title`, `_yoast_wpseo_metadesc`, `_yoast_wpseo_focuskw` (REST API로 POST 가능, show_in_rest 활성화 필요)
- **미디어 업로드**: `POST /wp-json/wp/v2/media`

### 2.6 티스토리

#### 콘텐츠 포맷
- **에디터**: 기본 에디터 + HTML 직접 편집
- **목차(TOC)**: H2(Heading 1), H3(Heading 2), H4(Heading 3) 태그로 자동 생성
- **광고 배치**: AdSense 자동 삽입 (본문 상단/중간/하단)
- **스킨**: HTML/CSS 완전 커스터마이징 가능

#### SEO 규칙
| 항목 | 규격 |
|------|------|
| **제목** | 30-50자 (키워드 앞 배치) |
| **본문** | 1,500-3,000자 |
| **이미지** | 최소 3-5장, 대체 텍스트 필수 |
| **태그** | 최대 10개 (포스트당) |
| **카테고리** | 계층 구조 가능 |

#### API 연동
- **상태**: **Open API 완전 종료** (2024년 2월)
- **글 작성/수정**: API 불가
- **이미지 첨부**: API 불가
- **대안**: Selenium/Playwright 기반 자동화 또는 수동 발행
- **발행 제한**: 일 5건 제한 (패턴 감지 시)

### 2.7 브런치

#### 콘텐츠 규격
| 항목 | 규격 |
|------|------|
| **제목** | 최대 20자 (10자 이내 권장) |
| **부제목** | 최대 40자 |
| **본문** | 공식 제한 미공개 (실질적 2,000-4,000자 최적) |
| **이미지** | 940px 이상 와이드, 한 줄 최대 3장 |
| **커버 이미지** | 960x540px 이상 권장 |

#### 에디터 특성
- 자체 에디터 (외부 HTML 삽입 불가)
- 글꼴/크기/색상 제한적 변경 가능
- 인용, 링크, 구분선, 이미지, 동영상 삽입
- 맞춤법 검사 기능 내장
- **마크다운 미지원**

#### 매거진 구조
- 태그 최대 3개 (매거진 단위)
- 글 → 매거진 편입 가능 (다중 매거진 소속 가능)
- 브런치북: 여러 글을 모아 전자책 형태로 발행

#### API 연동
- **상태**: **공개 API 없음**
- 글 발행은 웹 에디터를 통해서만 가능
- **작가 승인제**: 신청 후 심사 통과 필요
- **대안**: 수동 발행만 가능

### 2.9 뉴스레터 (이메일)

#### 핵심 규격
| 항목 | 권장 규격 |
|------|----------|
| **제목(Subject)** | 30-45자 (모바일 기준, 데스크톱 60자) |
| **프리헤더** | 35-55자 (모바일 50자 이후 잘림) |
| **본문 길이** | 200단어 이내 (스캔 가능한 구조) |
| **CTA 버튼** | 1개 주요 CTA (고대비 색상) |
| **이미지 너비** | 600px 권장 (최대 800px) |
| **전체 너비** | 600-800px (고정 너비 레이아웃) |

#### HTML 호환성 규칙
- **인라인 CSS만 사용** (외부 스타일시트 미지원 클라이언트 다수)
- `<style>` 태그: Gmail 지원, Outlook 일부 미지원
- Flexbox/Grid: **사용 금지** (테이블 레이아웃 사용)
- 웹폰트: 대부분 미지원 (시스템 폰트 폴백 필수)
- 다크 모드: `prefers-color-scheme` 미디어 쿼리 (제한적 지원)
- 숨겨진 프리헤더: `display:none` div + `&nbsp;` 버퍼 기법

#### 2026 트렌드
- "하이브리드 플레인텍스트" 디자인 (개인 이메일처럼 보이되 단일 CTA 포함)
- 프리헤더 있는 이메일: 오픈률 22.3% vs 없는 이메일 19.3%

#### API 연동 (주요 ESP)
| ESP | 엔드포인트 | 발송 제한 |

## 3. AI 콘텐츠 생성 시 프롬프트 핵심 규칙

### 3.1 공통 규칙 (모든 채널)

```
[공통 제약사항]
1. 각 채널의 글자수 제한을 절대 초과하지 않을 것
2. 첫 문장(훅)에 핵심 메시지를 담을 것 — 대부분의 채널에서 접힘(더보기) 처리됨
3. CTA(Call to Action)는 본문 말미에 명확히 배치
4. 이모지 사용은 채널 성격에 맞게 조절 (인스타: 자유, 링크드인: 절제, 뉴스레터: 최소)

```
